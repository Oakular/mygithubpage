----- First lets clear the confusion:

---- Difference between Git and Github:

  • Git is the Version Control Software
  • Github is a website to push and pull source code to/from
  • Github provides us with a central server to use Git with

---- Git works on all 3 major OSes

For UNIX systems (Linux, macOS) install Git via the package manager for your system. The website link above suggests downloading git directly for Mac. I suggest using Homebrew instead.
    On Ubuntu: sudo apt-get install git-all
    On macOS (with Homebrew): brew install git
    paste the commands into your terminal
On Windows download the executable file from the website above.

---- I recommend using the terminal commands

Why?

  • Once you have learnt them, they are faster to use
  • There is no official Git GUI and in industry not every company uses GitHub. Just knowing their GUI is not good practice


----- MAJOR GIT COMMANDS -----

    git init : when you run this command in your current directory it will add that directory (and any sub-directories) to be tracked by git git add : this command tells git to "track" a file. This means git will become aware of any changes to the file and will stage them ready for "commit"

    git commit | -a | -m : this command commits a change that you have made. it is important to note that this does NOT upload the change to Github.
    It just locks in the change you have made so that the next time you push to Github this commit will be uploaded with that push
      - The -a flag tells git to add any changed files before making a commit (it skips the use of the add command shown above)
      - The -m command opens a text editor to add a message alongside a commit. It is good practice to leave a message with every commit to detail it


    git push git-server-name branch-name : this command will push any commits you have made to Github.
    This is the command that uploads those changes to the central repository

    git pull git-server-name branch-name : the pull command syncs your local repository with the one hosted on Github.
    This command is used to make your version of the source code up to date

If you understand these four commands you can use Git.
